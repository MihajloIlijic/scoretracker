ZWISCHENBERICHT - Score Tracker Projekt
========================================

BESCHREIBUNG (Was, Warum, Wie)
-------------------------------

Was:
Das Score Tracker Projekt ist eine Webanwendung zur Verwaltung von Spielständen 
und Turnieren. Die Anwendung besteht aus einem Flutter Web Frontend und einem 
Go Backend, die über eine REST API kommunizieren. Als Datenbank wird PostgreSQL 
verwendet, und die gesamte Anwendung wird mit Docker containerisiert.

Warum:
Ziel ist es, eine moderne, skalierbare Webanwendung zu entwickeln, die es 
ermöglicht, Spielstände zu verwalten und Turniere zu organisieren. Durch die 
Verwendung von Flutter Web kann eine konsistente Benutzeroberfläche erstellt 
werden, während Go als Backend-Sprache für hohe Performance und einfache 
Wartbarkeit sorgt. Docker ermöglicht eine einfache Deployment- und 
Entwicklungsumgebung.

Wie:
Die Anwendung wurde als Microservice-Architektur aufgebaut:
- Frontend: Flutter Web für die Benutzeroberfläche
- Backend: Go mit Gin Framework für die REST API
- Datenbank: PostgreSQL für die Datenspeicherung
- Containerisierung: Docker Compose für die Orchestrierung aller Services

Die Kommunikation zwischen Frontend und Backend erfolgt über HTTP REST API 
Endpoints. Das Frontend sendet Requests an das Backend, welches die Daten 
in der PostgreSQL Datenbank speichert oder abruft.
